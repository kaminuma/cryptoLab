# 古典暗号カタログ

公開版 CryptoLab を「百科事典＋実験場」として育てるため、主要な古典暗号を整理した。時代背景・方式・手順・弱点に関する記述は、各種公開資料（暗号史の文献、専門サイト、博物館アーカイブなど）をベースにした要約である。

| 暗号 | 時代・背景 | アルゴリズム概要 | 特徴・弱点 |
| --- | --- | --- | --- |
| **シーザー暗号 (Caesar)** | 紀元前 1 世紀。ユリウス・シーザーが軍通信で使用。 | アルファベットを一定量シフトする単純換字。`C = (P + s) mod 26` | 総当たりが 26 通り。頻度分布がそのまま残るため現代では教育用途。 |
| **アトバッシュ (Atbash)** | 古代ヘブライ語圏。宗教文書にも登場。 | A↔Z、B↔Y のようにアルファベットを逆順で置換。 | 固定換字のため自動復号が容易。実装はシンプル。 |
| **スキュタレー (Scytale)** | 古代ギリシャ・スパルタ軍が使用した円筒暗号。 | 棒に帯を巻きつけて文字を書き、帯を外すと転置になる。棒の直径が鍵。 | 転置のみなので頻度分布は変化せず、棒サイズを推測すれば解読可能。 |
| **ヴィジュネル暗号 (Vigenère)** | 1550 年頃、ベルジュの著作で紹介。 | キーワードで周期的にシフトを切り替える多表式換字。`C = (P + K_i) mod 26` | 単純頻度分析に強いが、キー周期推定（カシスキー／フリードマン）で破られる。 |
| **オートキー暗号 (Autokey)** | 16 世紀。ヴィジュネルの改良案。 | キーワードに平文（または直前の暗号文）を連結し、シフト値の周期を長くする。 | キー再利用パターンが解析されやすく、連分布解析で崩される。 |
| **プレイフェア暗号 (Playfair)** | 1850 年代。チャールズ・ホイートストンが考案。 | 5×5 グリッド（二文字ペア）で換字。`I/J` を統合して 25 文字化。 | 単文字頻度には強いが二文字頻度で解析可能。第一次世界大戦でも使用。 |
| **Hill 暗号 (Hill cipher)** | 1929 年。レスター・S・ヒルが発明。 | 線形代数を用いたブロック換字。`C = K·P mod 26`。 | 行列の逆行列が必要。既知平文攻撃で行列を復元されやすい。 |
| **ワンタイムパッド (OTP)** | 1917 年、バーロー＆メイジャー。 | 真にランダムな鍵ストリームと平文を XOR。 | 鍵を一度しか使わず完全乱数なら情報理論的安全。鍵配布が最難関。 |
| **エニグマ (Enigma)** | 1920〜40 年代のドイツ軍。 | ローター、反射板、プラグボードによる多段換字。 | 自己写像禁止など構造的クセあり。ブレッチリーパークで解読（ボンベ等）。 |

## 実装視点の分類

- **換字式 (Substitution)**  
  アルファベットの置き換え。シーザー、アトバッシュ、ヴィジュネル、プレイフェアなど。
- **転置式 (Transposition)**  
  文字の並びだけ入れ替える方式。スキュタレーや二重転置暗号など。頻度分布は保持される。
- **多表式 (Polyalphabetic)**  
  複数の換字表を切り替える。ヴィジュネルやオートキーが該当。
- **ブロック線形**  
  Hill 暗号のようにベクトルと行列を使い、まとめて換字する方式。
- **ローター式 / 電気機械式**  
  エニグマのように物理ローターで換字を複雑化。日替わりキーやプラグ設定で状態が変わる。
- **完全秘匿系**  
  ワンタイムパッドのように数学的に証明された完全秘匿性を持つもの。

## 実装ロードマップ

1. `src/data/classical.ts` から UI にデータを供給し、カタログ → 詳細 → デモの順で差し替え表示できるようにする（完了）。
2. 実装済みのシーザー / ヴィジュネル / アトバッシュに続いて、プレイフェア・Hill・スキュタレーなどを順次追加する。
3. 各暗号の説明テキストは本ドキュメントをソースにしつつ、Web UI 側でも要約・参考リンクを表示する。
4. 実装が難しい暗号（エニグマ等）は視覚的なダイアグラムや外部資料リンクを中心にし、操作系 UI を別途計画する。
5. 史実・研究史の要約は定期的に見直し、公的資料や研究論文を参照してアップデートする。
